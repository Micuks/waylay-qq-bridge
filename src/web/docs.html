Overview

What Waylay is

Waylay is a lightweight headless NTQQ bridge written in plain JavaScript with built-in OneBot v11 and Milky protocol support. It loads QQ's wrapper.node native module directly — ideal for minimal deployments with low resource requirements.

Highlights

  • 13 source files, ~4,000 lines, 1 runtime dependency (ws)
  • Sub-millisecond read responses from an event-driven in-memory cache
  • Non-blocking I/O for media downloads and audio/video probing
  • Docker one-liner, or bare-metal via systemd
Getting started

Quickstart

Run the latest image from Docker Hub, scan the QR code, and connect your bot framework.

1. Start the container

bash
$ docker run -d --name waylay --privileged \
    -p 13000:13000 -p 3001:3001 \
    micuks/waylay:latest

2. Scan to log in

Visit http://localhost:13000/qrcode and scan with QQ Mobile.

3. Point your framework at :3001

Configure a forward WebSocket in Koishi (adapter-onebot) or a reverse WebSocket in Yunzai. Set ONEBOT_WS_REVERSE_URLS to skip exposing a port.

log
[server] Listening on 0.0.0.0:13000
[bridge] kernel up
[onebot] WS forward ready · :3001
[onebot] client connected
Install

Installation

Three supported paths: Docker Hub image, bare-metal install script, or build from source.

Docker Hub (recommended)

bash
$ docker run -d --name waylay --privileged \
    -p 13000:13000 -p 3001:3001 \
    -e AUTO_LOGIN_QQ=123456789 \
    -v waylay_qq:/root/.config/QQ \
    micuks/waylay:latest

Bare-metal Linux (amd64)

bash
$ curl -fsSL https://raw.githubusercontent.com/Micuks/waylay-qq-bridge/master/install.sh | sudo bash
$ sudo systemctl enable --now waylay

From source

bash
$ git clone https://github.com/Micuks/waylay-qq-bridge.git
$ cd waylay-qq-bridge
$ docker compose up -d --build

Protocols

OneBot v11

Forward WebSocket on :3001, reverse WebSocket via ONEBOT_WS_REVERSE_URLS. Response envelope follows the OneBot v11 standard.

Connection

text
# Forward:
ws://host:3001/
# Reverse:
ONEBOT_WS_REVERSE_URLS=["ws://host.docker.internal:2536/OneBotv11"]

Reference

Action API

OneBot v11 actions. All actions return the standard OneBot envelope.

send_group_msg message
Send a message to a group. Auto-detects segment types.
get_group_list group
Return cached list of groups. Sub-ms response.
set_group_ban group
Mute a group member for N seconds.
upload_group_file file
Upload a file to a group folder. Async; does not block.
delete_msg message
Recall a message by id.
